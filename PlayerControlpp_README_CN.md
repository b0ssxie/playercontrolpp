## 简介

PlayerControl++ 是一个纯客户端的 Fabric 模组，旨在增强原版玩家的操作体验。它提供自动前进、快速转向、带导航点的可配置路径流系统，以及基于 RLE 压缩的高保真输入录制与回放功能。模组同时集成了容器缓存自动化、自动注水辅助与自动材料收集，与 Litematica、Baritone 等工具深度联动。

所有操作均通过模拟玩家按键输入实现，不直接修改实体位置或发送异常数据包，尽可能兼容多人服务器与反作弊环境。

## 快速开始

1. 安装 Fabric Loader、Fabric API 和 MaLiLib。
2. 将本模组放入 `mods` 文件夹。
3. 进入游戏后按默认热键 `P + C` 打开配置界面，也可以从 Mod Menu 进入。
4. 在配置界面中为各功能绑定热键，按需开启自动前进、路径流、录制或缓存功能。

## 核心功能

### 自动前进

- 可自定义 Toggle 热键，按下后玩家持续向前移动，模拟按住 W 键。
- 切换世界或维度时自动关闭，ActionBar 状态提示。
- 轻量实现，不影响其他按键操作。

### 快速转向

- 可自定义 Trigger 热键，按下瞬间旋转指定角度（默认 180°，范围 0-360°）。
- 角度可在配置 GUI 中调整。

### 路径流系统

路径流系统是模组的核心功能之一，允许玩家预设多条带导航点的路径并自动行走。

**路径管理**
- 每条路径支持多个中间导航点（起点 → 途经点 → 终点）。
- 支持添加 / 删除路径、添加 / 删除导航点、动态热键绑定。
- 路径数据以 JSON 格式持久化存储。

**路径执行**
- 模拟按住 W 键前进 + 自动 yaw 视角修正，模拟真人行走。
- 混合转向模式：允许玩家轻微移动视角，偏差过大时自动平滑修正。
  - 分级修正速度（>45° = 25°/tick, >15° = 18°/tick, 默认 15°/tick）。
  - 死区 ±2° 内不修正，避免"锁头"感。
- 仅控制 XZ 平面，忽略垂直偏差。

**循环模式**

| Loop Count | 行为 |
|------------|------|
| 1 | 单次：起点 → 终点，结束 |
| >1 | 往返 N 次：起点 → 终点 → 起点（往复） |
| 0 | 无限循环 |

**卡住检测**
- 3 秒未移动 → 自动跳跃一次。
- 跳跃后 5 秒仍未移动 → 终止路径，ActionBar 提示。

**安全系统**
- 世界切换、维度变更、玩家死亡、断开服务器 → 立即停止。

**路径级别选项（每条路径独立配置）**
- **Sprint（疾跑）**：开启后执行路径时全程按住疾跑键。
- **LayerCtrl（图层控制）**：开启后每次遍历完成自动切换 Litematica 投影渲染层。

### 玩家行为录制与回放

以 Tick 级精度录制玩家输入并回放，支持 RLE 压缩和位置修正，v1.4 起采用二进制 NBT 格式。

**录制内容**
- 移动（WASD）、疾跑、跳跃、潜行、视角（Yaw + Pitch）、左右键操作。

**RLE 压缩**
- 连续相同输入合并为 Segment，每段有 duration 字段表示持续 Tick 数。
- 大幅减少文件大小，同时保持回放精度。

**NBT 二进制格式（v1.4 重构）**
- 录制文件由 JSON（.json）改为 NBT 二进制格式（.pcr 扩展名）。
- 录制时使用 RingBuffer 分块预分配数组（CHUNK_SIZE=1024），减少 GC 压力。
- 录制停止后数据在后台线程（PCpp-RecSave）异步写入 .pcr 文件，不阻塞主线程。
- 损坏文件检测：异常的 .pcr 文件加载时在 ActionBar 显示错误提示。
- 始终记录位置关键帧（每 20 Tick），回放时位置偏差 >0.2 块自动修正至录制轨迹。

**回放功能**
- Play Count = 0 无限循环，Play Count = N 重复回放 N 次。
- 回放开始 / 循环时模拟行走至录制起点，不使用瞬移。
- 按需加载录制文件（用户点击播放时才读取完整数据）。

**文件管理**
- 索引文件 + 独立录制文件分离存储。
- GUI 录制列表仅读取索引，打开速度与录制数量无关。
- 删除操作仅删除对应录制文件 + 更新索引。

**安全机制**
- 播放时禁止开始新录制。
- 录制 / 播放开始时自动退出所有 GUI 界面。
- 录制期间 ActionBar 常驻提示。

### Litematica 投影联动

路径执行过程中，每次完成一次遍历（起点→终点 或 终点→起点），自动切换 Litematica 投影的 SINGLE_LAYER 渲染层。

- 反射调用，无编译时依赖，Litematica 未安装时静默失败。
- 仅在 SINGLE_LAYER 模式下生效。
- 支持自定义图层增量（正数向上 / 负数向下）。

### 自动缓存附近容器

帮助 ChestTracker 等容器缓存类模组自动获取附近容器内容，无需手动逐个打开容器。

- 以玩家位置为中心、游戏模式最大交互距离为半径，球形扫描未缓存容器。
- 6 状态任务状态机驱动：SCANNING → OPENING_CONTAINER → WAITING_AFTER_OPEN → CLOSING_GUI → COOLDOWN → AUTO_STOP_COUNTDOWN。
- 支持自定义可缓存容器白名单（默认包含全部 28 种原版存储容器）。
- 可配置缓存容器延迟（1-200 Tick，默认 1 Tick），首个容器立即打开，不影响 3 秒自动停止。
- 按距离从近到远排序，已缓存容器不重复打开。
- 玩家移动时持续重新扫描，新容器自动加入流程。
- 所有操作通过正常客户端交互流程完成，兼容单人、局域网和多人服务器。
- 世界切换时立即停止。

### 自动注水

自动检测 Litematica 投影中可含水方块并使用水桶进行含水操作，需要 Litematica 投影已加载。

- 6 状态任务状态机控制，在玩家可达距离内扫描投影中的可含水方块。
- 自动在快捷栏/背包中寻找水桶并切换至手持。
- 自动旋转视角面朝目标方块，通过 interactBlock + interactItem 模拟右键含水操作。
- 新热键"自动注水"（默认无绑定），按下开启/关闭。
- 可配置扫描半径（0-5 格）和操作延迟（1-200 Tick）。
- 安全机制：潜行时暂停，死亡/世界切换时自动停止。

### 自动材料收集

基于 Baritone + ChestTracker 从 Litematica 投影自动收集所需材料。

- 需要 Baritone 和 ChestTracker 模组。
- 自动从附近已缓存容器中提取投影所需材料。
- 背包满时自动存储至潜影盒。

## 配置分类

配置界面（`P + C` 打开）分为以下标签页：

| 标签页 | 内容 |
|--------|------|
| **Hotkeys** | 通用热键：打开配置界面、自动前进、快速转向、切换录制、自动缓存附近容器、自动注水 + 可缓存容器白名单 |
| **Route Hotkeys** | 动态路径热键（每个路径独立绑定） |
| **Settings** | 转向角度、缓存容器延迟、注水扫描半径、注水操作延迟 |
| **Routes** | 路径流系统编辑（路径列表 / 导航点管理 / 选项开关） |
| **Recording** | 录制与回放（录制管理 / 播放控制 / 损坏文件检测） |
| **Baritone** | 自动材料收集配置 |

## 依赖与兼容

**必需依赖**

| 模组 | 最低版本 |
|------|----------|
| Fabric Loader | 0.17.3+ |
| Fabric API | 0.119.2+ |
| MaLiLib | 0.23.5+ |

**可选联动**

| 模组 | 用途 |
|------|------|
| ModMenu | 模组配置界面入口（建议安装） |
| Litematica | 投影渲染层切换 / 自动注水联动（反射调用，无编译时依赖） |
| ChestTracker | 自动缓存附近容器 / 材料收集联动（反射调用，无编译时依赖） |
| QuickShulker | 快捷打开背包中的潜影盒（反射调用，无编译时依赖） |
| Baritone | 自动材料收集路径执行（反射调用，无编译时依赖） |

> [!NOTE]
> 所有联动模组通过反射调用实现，无编译时依赖。缺失时对应功能静默降级，不影响模组正常使用。
